## 📌 문제 핵심 아이디어
- 장르별 총 재생수 기준으로 내림차순
- 각 장르 내에서는 재생수(내림차순) → 고유번호(오름차순) 다중 조건 정렬
- 장르당 최대 2개의 곡만 인덱스 형태로 추출하여 반환

## ⚠️ 내가 놓친 부분 / 틀린 이유
- 파이썬 문법 부족

## 💡 핵심 Python 문법/팁

- 파이썬 딕셔너리를 Value(값) 기준으로 정렬하고 싶을 때

```python
#오름차순
sorted_dict = dict(sorted(my_dict.items(), key=lambda x:x[1]))

#내림차순

sorted_dict = dict(sorted(my_dict.items(), key=lambda x:x[1], reverse = True)

#my_dict.items() : (Key, Value) 튜플 형태의 쌍을 추출한다.

#key=lambda x: x[1] : 튜플의 1번 인덱스인 값을 기준으로 정렬 지정

#dict(…) : 정령되어 반환된 튜블 리스트를 (1,2) 딕셔너리 객체로 변환.
```

- 파이썬에서는 객체를 `.` 으로 접근할 수 없음.

```python
info = {
'type' : genres[i],
'plays' : plays[i],
'idx':i
}

info['type'] = genres[i]
info['plays'] = plays[i]
info['idx'] = i 
```

- `lambda` 와 튜플을 활용한 다중 조건 정렬

단일 기준이 아니라 우선순위가 여러 개일 때, `-` 부호로 내림차순/오름차순을 조합함

```python
# 내림차순 (-), 오름차순(+)
filtered_list_sorted = sorted(filtered_list, key=lambda x:(-x['plays'],x['idx'])))
```

- `append()` vs `extend()` 차이

리스트 내부 요소를 flattening(평탄화)해서 1차원 배열로 넣을 때는 `extend()` 사용

| **메서드** | **예시 연산** | **결과 (answer)** | **비고** |
| --- | --- | --- | --- |
| `append()` | `answer.append([1, 2])` | `[[1, 2]]` | 리스트 통째로 삽입 (2차원) |
| **`extend()`** | **`answer.extend([1, 2])`** | **`[1, 2]`** | **요소만 펼쳐서 추가 (1차원)** |

### 코드

```python
def solution(genres, plays):
    answer = []

    genres_dict= {}

    # 1. generes랑 plays랑 매핑하여 객체 생성
    for i in range(len(genres)):
        if genres[i] in genres_dict.keys():
            genres_dict[genres[i]] += plays[i]
        else:
            genres_dict[genres[i]] = plays[i]

    # 재생수 많은 장르 정렬
    sorted_dict = dict(sorted(genres_dict.items(),key=lambda x:x[1], reverse=True))

    # sorted_dict에서 하나의 장르 중 노래 더 많은거 -> 고유번호 낮은거 
    playlist_dict = []
    for i in range(len(genres)):
        info = {}
        info['genres']=genres[i]
        info['plays'] = plays[i]
        info['idx'] = i
        playlist_dict.append(info)

    for target_genres in sorted_dict:
        filtered_list = [item for item in playlist_dict if item['genres'] == target_genres]
        filtered_list_sorted = sorted(filtered_list,key=lambda x:(-x['plays'],x['idx']))
        # print('1',filtered_list,filtered_list_sorted,[item['idx'] for item in filtered_list_sorted][:2])

        answer.extend([item['idx'] for item in filtered_list_sorted][:2])
        

    return answer
```

### 아쉬운 부분

1. `dict.keys()` 탐색의 비효율성

```python
if genres[i] in genres_dict.keys():
```

- 위와 같이 쓰면 딕셔너리의 키 리스트를 별도로 조회하므로 $O(N)$시간이 걸림. 
딕셔너리 자체에 `in` 을 쓰면 $O(1)$ 로 즉시 확인됨.

1. 불필요한 전체 딕셔너리 재정렬 (`dict(…)`)

```python
sorted_dict = dict(sorted(genres_dict.items(), key=lambda x:x[1], reverse=True))
```

- sorted()의 결과는 이미 **`[('pop', 3100), ('classic', 1450)]`** 형태의 정렬된 튜플 리스트로 반환된다.
이를 굳이 다시 `dict()` 로 반환할 필요 없이, 정렬된 리스트에서 바로 장르 이름만 꺼내어 루프를 돌리는것이 메모리와 연산 측면에서 좋다.

```python
# 개선 : 튜플 리스트에서 장르 이름만 바로 순회 
sorted_genres = sorted(genres_dict.items(), key=lambda x:x[1], reverse=True)
```

1. 리스트 순회 기반 필터링 ($O(N^2)$ 가능성)

```python
# 기존 코드
for target_genres in sorted_dict:
    filtered_list = [item for item in playlist_dict if item['genres'] == target_genres]
```

- 장르가 $M$개 있고 총 노래가 $N$개일 때, 루프를 돌 때마다 `playlist_dict` 전체($N$개)를 처음부터 끝까지 다 훑으며 조건에 맞는 곡을 찾습니다. 즉, 전체 시간 복잡도가 $O(M \times N)$이 됩니다.
`playlist_dict`를 단순 리스트가 아니라 **"장르를 Key로 가지는 딕셔너리"** 형태로 처음부터 그룹화해 두면, 장르별 곡을 찾을 때 **$O(1)$** 만에 바로 접근할 수 있습니다.

### 개선된 코드

```python
from collections import defaultdict

def solution(genres, plays):
    answer = []

    genres_plays = defaultdict(int)      # 장르별 총 재생수 저장
    songs_by_genre = defaultdict(list)    # 장르별 곡 정보 저장 {장르: [(plays, idx), ...]}

    # 1. 한 번의 루프(O(N))로 두 구조를 동시에 생성
    for i, (genre, play) in enumerate(zip(genres, plays)):
        genres_plays[genre] += play
        songs_by_genre[genre].append((play, i))

    # 2. 재생수가 많은 장르 순으로 정렬
    sorted_genres = sorted(genres_plays.keys(), key=lambda g: genres_plays[g], reverse=True)

    # 3. 정렬된 장르 순서대로 상위 2개 곡 추출
    for genre in sorted_genres:
        # 재생수 내림차순(-), 인덱스 오름차순(+) 정렬
        sorted_songs = sorted(songs_by_genre[genre], key=lambda x: (-x[0], x[1]))
        
        # 상위 최대 2개의 인덱스(x[1])만 추출하여 answer에 추가
        answer.extend([idx for play, idx in sorted_songs[:2]])

    return answer
```

🌟 개선된 코드의 장점

1. **`zip` & `enumerate` 활용:** `range(len(genres))` 대신 파이썬다운 방식으로 인덱스와 요소를 한 번에 꺼냅니다.
2. **`defaultdict` 활용:** `if genre in dict`와 같은 예외 처리 구문 없이 바로 누적 연산을 수행합니다.
3. **탐색 시간 단축:** 장르별로 이미 곡들이 묶여 있으므로, `if item['genres'] == target_genre`로 전체 데이터를 검색할 필요가 없어 성능이 훨씬 빨라집니다.

```py
from collections import defaultdict

def solution(genres, plays):
    answer = []

    # 장르별 총 재생수
    genres_plays = defaultdict(int)
    # 장르별 곡 정보 저장
    songs_by_genres = defaultdict(list)

    # 1. 객체 세팅
    for i,(genre,play) in enumerate(zip(genres,plays)):
        genres_plays[genre]+=play
        songs_by_genres[genre].append((play,i))
    
    # 2. 재생수가 많은 수대로 정렬
    sorted_genres = sorted(genres_plays.keys(),key=lambda x:genres_plays[x],reverse=True)


    # 3. 정렬된 장르 순서대로 상위 2개의 곡만 추출
    for genre in sorted_genres:
        # 재생수 내림차순, 고유번호 오름차순
        sorted_songs = sorted(songs_by_genres[genre], key=lambda x:(-x[0],x[1]))
        # 상위 2개 answer추가
        answer.extend([idx for _,idx in sorted_songs[:2]])

    return answer
```

- `enumerate`가 어떻게 `range(len())`을 대신할 수 있나?
인덱스(숫자)를 직접 계산해서 접근하느냐, 알아서 만들어주느냐 차이이다.

`enumerate`는 반복 가능한 객체 옆에 0부터 시작하는 순번(인덱스)를 자동으로 붙여주는 역할을 한다. 
1. `zip(genres, plays)` 가 장르수와 재생수 짝지어준다. : `('classic',500)`
2. `enumerate(...)`가 순번을 붙여줌 : `(0,('classic',500))`
따라서 range(len())처럼 genres[i]로 다시 접근할 필요없이, 인덱스와 실제 값들을 한번에 변수로 받아올 수 있다.

| 구분 | `range(len(genres))` | `enumerate(zip(genres, plays))` |
| :--- | :--- | :--- |
| **반환값** | 숫자 `0, 1, 2...` 만 생성 | `(인덱스, (장르, 재생수))` 묶음 생성 |
| **값 접근** | `genres[i]`, `plays[i]` 직접 지칭 | `i`, `genre`, `play` 변수로 즉시 사용 |
| **가독성** | 대괄호`[]`가 많아 코드가 지저분해짐 | 변수 이름이 명확하여 가독성이 좋아짐 |
