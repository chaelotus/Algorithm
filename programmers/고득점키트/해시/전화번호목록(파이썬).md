## 📌 문제 핵심 아이디어
- 이 문제는 phone_book의 길이는 1 이상 1,000,000 이하이기 때문에 O(N log N)이나 O(N X L) 시간복잡도로 문제를 풀어야 한다.

## ⚠️ 내가 놓친 부분 / 틀린 이유
- 처음에 이중 for문으로 접근했다가 시간 초과($O(N^2)$) 발생.

## 💡 핵심 Python 문법/팁
- sorted() : 원본 유지하고 새로운 리스트 반환
- sort() : 원본을 직접 변경
- startswith() : 문자열이 특정 접두사로 시작하는지 여부 확인(True/False)

### 문제 해결 방법

- 정렬 방법으로 문제 해결
- 정렬을 하면 내 근처에 접두어가 같은 것이 무조건 옴. 근처에 접두어가 같은 숫자가 없다는 것은 접두어인 숫자가 없다는 것.
- 따라서 다음 숫자만 확인하면 이중 for문을 사용하지 않아도 됨.

### 코드

```python
def solution(phone_book):
    
    sorted_phone_book = sorted(phone_book)

    for i in range(len(sorted_phone_book)):
        if i == len(sorted_phone_book)-1:
            break
        if sorted_phone_book[i+1].startswith(sorted_phone_book[i]):
            return False
    return True
```

- 현재 나의 코드의 시간복잡도는 O(N log N X L)이다.
- 왜 이런 시간 복잡도가 나오나?
    - `sorted()` 는 Timsort 알고리즘을 사용하므로 기본적으로 O(N log N)번 비교한다.
    - 하지만 문자열을 비교할 때는 두 분자열의 길이만큼 또 비교해야한다.
    - 따라서 O(N log N * L)이 된다.
