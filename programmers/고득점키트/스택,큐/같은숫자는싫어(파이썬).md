### 코드
```py
def solution(arr):

    # stack에서 제일 최근값이랑 들어오려는 수랑 같으면 그냥 넘기기
    # 다르면 push

    stack = []
    arr_len = len(arr)

    for i in range(arr_len):
        if len(stack)>0 and stack[len(stack)-1] == arr[i]:
            continue
        
        stack.append(arr[i])
    
    
    return stack
```

### 아쉬운 점

- 음수 인덱스 미활용
    - 파이썬은 리스트의 마지막 원소를 가리킬 때 `len(stack)-1`  같은 계산 없이 `stack[-1]` 로 접근할 수 있는 문법을 제공함.
- `range(len(arr))` 사용
    - `i` 가 필요한 게 아니라 원소값 자체가 필요할 때는 `for num in arr` 로 직접 순회하는것이 훨씬 직관적이다.
- stack이 비어져 있지 않을 때는 `if stack and` 로 대체 가능 ⇒ `stack[-1:]` 을 사용하면 if stack 생략 가능.

**슬라이싱**을 사용하면 (stack[-1:]) 빈 리스트일 때 **IndexError**가 나지 않고 빈 리스트 []를 반환한다.

그래서 stack이 비었는가를 따로 체크하지 않고 num과 비교할 수 있어 코드가 간결해진다.
그래서 stack이 비었는가를 따로 체크하지 않고 [num]과 비교할 수 있어 코드가 간결해진다. 

| **구분** | **num** | **[num]** |
| --- | --- | --- |
| **자료형** | 정수형 (`int`) | 정수가 하나 들어있는 **리스트 (`list`)** |
| **형태** | `1` | `[1]` |
| **비유** | 사과 알맹이 한 개 | 사과 한 개가 들어있는 **상자** |

```python
stack = [1, 2, 3]

# 1. 슬라이싱의 결과는 항상 '리스트'입니다.
print(stack[-1:])  # 결과: [3] (숫자 3이 아니라 [3]이라는 리스트!)

# 2. 따라서 비교할 대상도 '리스트' 모양으로 맞춰주어야 합니다.
# [3] == 3       -> False (타입이 달라서 비교 불가)
# [3] == [3]     -> True  (타입이 같아 정상 비교 가능!)
```

### 개선된 코드
```py
def solution(arr):

    stack = []

    for num in arr:
        if stack[-1:] != [num]:
            stack.append(num)
    
    return stack
```
